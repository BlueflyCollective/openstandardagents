# OSSA Visual Identity Guidelines
## Open Standards for Scalable Agents

### Logo & Wordmark

#### Primary Logo
The OSSA logo combines geometric precision with organic flow, representing the intersection of structured standards and adaptive agent intelligence.

**Logo Components:**
- **Symbol**: Interconnected hexagonal nodes forming a scalable network pattern
- **Wordmark**: "OSSA" in custom geometric typography
- **Tagline**: "Open Standards for Scalable Agents" in clean, modern sans-serif

#### Logo Variations
- **Primary**: Full color on light background
- **Reversed**: White on dark background
- **Monochrome**: Single color applications
- **Icon Only**: Symbol without wordmark for small applications
- **Horizontal**: Side-by-side arrangement for wide formats
- **Stacked**: Vertical arrangement for square formats

#### Clear Space & Minimum Sizes
- Clear space: Minimum of 'O' height on all sides
- Minimum digital size: 120px wide
- Minimum print size: 0.75 inches wide
- Icon minimum: 32px x 32px

### Typography

#### Primary Typeface
**Inter** - A highly legible, open-source typeface designed for digital interfaces

**Weights & Usage:**
- **Black (900)**: Hero headlines
- **Bold (700)**: Section headers, emphasis
- **Semibold (600)**: Subheaders, navigation
- **Medium (500)**: Body emphasis, buttons
- **Regular (400)**: Body text
- **Light (300)**: Supporting text, captions

#### Monospace Typeface
**JetBrains Mono** - For code examples, technical documentation

#### Type Scale
```
Hero:        48px / 56px line-height
H1:          36px / 44px line-height
H2:          28px / 36px line-height
H3:          24px / 32px line-height
H4:          20px / 28px line-height
Body Large:  18px / 28px line-height
Body:        16px / 24px line-height
Small:       14px / 20px line-height
Caption:     12px / 16px line-height
```

### Visual Elements

#### Grid System
- **12-column grid** for web applications
- **8px baseline grid** for vertical rhythm
- **16px margins** on mobile
- **24px margins** on tablet
- **32px margins** on desktop

#### Iconography Style
- **Line weight**: 2px stroke
- **Style**: Geometric, minimal
- **Corner radius**: 2px
- **Grid**: 24x24px base
- **Scaling**: 16px, 24px, 32px, 48px variants

#### Illustration Principles
- **Isometric perspective** for technical diagrams
- **Flat design** with subtle gradients
- **Node-based** representations for agent networks
- **Data flow** visualizations using connecting lines
- **Consistent use** of brand colors

#### Photography Style
When photography is used:
- **Abstract technology** imagery preferred
- **Clean, minimal** compositions
- **Blue-toned** color grading
- **High contrast** with deep blacks
- **No people** in primary brand imagery
- **Server rooms**, circuit boards, abstract patterns

### UI Components

#### Buttons
- **Primary**: OSSA Blue background, white text
- **Secondary**: Transparent with OSSA Blue border
- **Tertiary**: Text only with underline on hover
- **Danger**: Signal Orange for destructive actions
- **Border radius**: 6px
- **Padding**: 12px 24px

#### Cards
- **Background**: White or Neutral 100
- **Border**: 1px solid Neutral 200
- **Border radius**: 8px
- **Shadow**: 0 2px 8px rgba(0,0,0,0.08)
- **Padding**: 24px

#### Code Blocks
- **Background**: Neutral 900
- **Text**: Syntax highlighting with brand colors
- **Border radius**: 6px
- **Font**: JetBrains Mono
- **Line numbers**: Neutral 500

#### Badges & Tags
- **Background**: Color at 10% opacity
- **Text**: Corresponding color at 100%
- **Border radius**: 4px
- **Padding**: 4px 8px
- **Font size**: 12px

### Motion & Animation

#### Principles
- **Purposeful**: Every animation has a function
- **Subtle**: 200-300ms duration for most transitions
- **Consistent**: Same easing curves throughout
- **Performance**: CSS animations preferred over JavaScript

#### Standard Easings
- **Ease Out**: cubic-bezier(0.0, 0.0, 0.2, 1)
- **Ease In Out**: cubic-bezier(0.4, 0.0, 0.2, 1)
- **Sharp**: cubic-bezier(0.4, 0.0, 0.6, 1)

- **Margins**: 1 inch all sides

### Brand Voice in Visual Context

#### Technical Diagrams
- Clean, precise line work
- Consistent iconography
- Clear labeling with Inter typeface
- Adequate white space for clarity

#### Marketing Materials
- Bold use of primary blue
- High contrast for impact
- Geometric patterns as backgrounds
- Focus on connection and scale

#### Presentation Templates
- Title slides with full-bleed color
- Content slides with generous white space
- Consistent header and footer elements
- Icon-based bullet points

### Accessibility Guidelines

#### Color Contrast
- **Normal text**: Minimum 4.5:1 contrast ratio
- **Large text**: Minimum 3:1 contrast ratio
- **UI elements**: Minimum 3:1 contrast ratio
- **Always test** with color blindness simulators

#### Typography
- **Minimum body text**: 16px
- **Line length**: 45-75 characters ideal
- **Line spacing**: 1.5x for body text
- **Avoid all caps** except for short labels

#### Interactive Elements
- **Minimum touch target**: 44x44px
- **Focus indicators**: Visible keyboard navigation
- **Alt text**: Required for all images
- **ARIA labels**: For icon-only buttons

### File Formats & Assets

#### Logo Files
- **SVG**: Primary format for all digital uses
- **PNG**: 2x resolution for raster needs
- **EPS**: Vector format for print
- **PDF**: For document embedding

#### Color Specifications
- **ASE**: Adobe Swatch Exchange
- **CSS**: Custom properties file
- **SCSS**: Variables file
- **JSON**: For application theming

### Usage Guidelines

#### Do's
- ✓ Maintain consistent spacing
- ✓ Use approved color combinations
- ✓ Follow the type hierarchy
- ✓ Ensure adequate contrast
- ✓ Keep the logo unmodified

#### Don'ts
- ✗ Stretch or distort the logo
- ✗ Use off-brand colors
- ✗ Create new logo variations
- ✗ Place logo on busy backgrounds
- ✗ Use outdated versions
